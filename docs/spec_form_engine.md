이 문서는 log\_templates의 JSON 데이터를 해석하여 화면을 구성하고 데이터를 수집하는 동적 폼 엔진의 작동 규격을 정의한다.



1\. JSON 스키마 구조 (양식 정의)

모든 양식은 아래의 핵심 필드를 포함한 JSON 형태로 저장된다.



fields: 입력 항목 리스트 (항목명, 입력 타입, 단위 등).



validation: 한계기준(CL) 설정. (예: min: 121, max: 125).



logic: 부적합(X) 발생 시 강제되는 후속 조치 정의.



2\. 입력 타입 및 판정 로직

체크형(Select): ○(적합) / ×(부적합).



수치형(Numeric): 직접 숫자 입력. 설정된 범위를 벗어나면 즉시 부적합으로 자동 판정.



텍스트형(Text): 특이사항이나 조치사항 기록용.



3\. 부적합 및 개선조치 (HACCP 핵심)

트리거: 판정 결과가 '부적합'일 경우, 화면에 즉시 \*\*\[부적합 사유]\*\*와 \[개선조치 사항] 입력 필드를 노출한다.



입력 제한: 부적합 항목이 있음에도 조치 사항을 적지 않으면 \[제출] 버튼을 비활성화하여 기록 누락을 원천 차단한다.



4\. 데이터 수집 및 저장

구조화: 수집된 데이터는 JSON.stringify()를 통해 DB의 data\_json 컬럼에 저장된다.



무결성: 저장 시점의 서버 타임스탬프를 함께 기록하여 사후 조작 가능성을 방지한다.
