HACCP 스마트 관리 시스템 v1.0 통합 마스터 기획안
1. 프로젝트 정의 및 v1.0 목표
명칭: HACCP 스마트 관리 시스템 v1.0

핵심 목표:

1공장(이유식), 2공장(장조림)의 표준 양식 헤더 및 결재 체계 수립

SQLite DB 스키마 확정 및 데이터 접근 모듈(Repository) 기초 구현

동적 폼 엔진의 최소 기능 구현 (JSON 읽기 -> 화면 렌더링 -> 데이터 수집)

2. v1.0 기술 스택 및 구조
Backend: Node.js + Express (v1.0 코어 구현)

Database: SQLite (초기 테이블 설계 및 Seed 데이터 구성)

디자인 패턴: Repository Pattern 기반의 DB 통신 격리

공장 식별: factory_id 필드를 통한 기초적인 데이터 구분

3. v1.0 세부 지침 (우선순위)
3.1 표준 헤더 및 결재 (v1.0 기준)
헤더: 모든 일지에 공통 적용될 문서번호, 개정번호, 작성자 정보 규격화

결재: 담당-검토-승인 3단 레이아웃 및 기본 상태값(작성중, 완료) 구현

3.2 동적 폼 엔진 기초
JSON 규격: 항목명, 입력타입(텍스트/숫자), 적합/부적합 체크 기능을 담은 JSON 스키마 확정

부적합 대응: 부적합(X) 클릭 시 비고란에 사유를 적게 하는 기초 로직

3.3 사용자 권한 기초
Users 테이블에 기초 계정 생성 및 factoryRoles를 통한 공장별 접근 제한 기초 구현

4. 개발 로드맵 (v1.0)
Step 1: DB 스키마 확정 및 seed.js를 통한 기초 데이터 삽입

Step 2: 표준 헤더 및 동적 폼 엔진 렌더링 검증

Step 3: 1공장(이유식) 대표 양식 1종, 2공장(장조림) 대표 양식 1종 이식 테스트
